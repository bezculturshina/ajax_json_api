JSON — стандарт обмена данными

JSON (JavaScript Object Notation) — текстовый формат обмена данными, основанный на JavaScript. Несмотря на это, он является независимым от языка и может использоваться практически везде. Например, многие программы, в том числе игры, используют json для хранения настроек, к примеру, VS Code – settings.json, в веб-разработке практически любой современный сайт получает данные в формате json, кроссплатформенные мобильные приложения используют json для обмена данными с сервером и так далее.

Как выглядят данные JSON на практике?

JSON поддерживает несколько базовых типов данных, которые позволяют описывать структуры любой сложности:

String — Текстовая строка в двойных кавычках.
Number — Целое число или число с плавающей точкой.
Boolean — Логическое значение: true или false.
Object — Набор пар "ключ: значение", заключенный в фигурные скобки { ... }.
Array — Упорядоченный список значений в квадратных скобках [ ... ].
Null — Пустое значение.

Преимущества и недостатки JSON

Преимущества

  • Легкий и компактный: Минимальный объем данных при передаче по сети.
  • Простой синтаксис: Легко читается человеком и быстро парсится машиной.
  • Универсальность: Поддерживается почти всеми языками программирования (JS, Python, PHP, Java и др.).

Недостатки

  • Нет функций: Передает только данные, не может содержать исполняемый код.
  • Без комментариев: Стандарт JSON не поддерживает комментарии внутри файла.
  • Объем больших данных: Для сверхсложных бинарных данных (видео, фото) лучше использовать другие форматы.
